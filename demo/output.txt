一. 基础开发工具
1. 软件源配置
软件源就相当于软件库，需要安装什么软件，只需要简单使用命令来进行安装。其中Ubuntu
系统中自带一个名为 apt 软件包管理工具，它主要负责 Ubuntu 系统中所有软件包的管理，包
括下载软件，卸载已安装的软件等。每当我们通过 sudo apt-get install <package-name>
去安装软件时，它就会从系统中的一个叫做源列表的文件去读取软件源的网址信息，查找包含
该软件的软件源网址，接着从上面把指定的软件下载下来。

OpenCV是一个开源的计算机视觉库，它包含了成百上千个封装好的图像处理和计算机视觉算

（1）Opencv安装
ubuntu22.04通过sudo apt 安装的opencv是4.10版本，由于该版本对nvidia jetson orin的
tensorrt加速无法提供完整支持，需要通过源码编译安装4.12版本的opencv

# 下载 OpenCV 4.12 源码
wget -O opencv.zip https://github.com/opencv/opencv/archive/4.12.0.zip
wget -O opencv_contrib.zip
https://github.com/opencv/opencv_contrib/archive/4.12.0.zip
unzip opencv.zip && unzip opencv_contrib.zip
# 编译安装
cd opencv-4.12.0
mkdir -p build && cd build
cmake -D CMAKE_BUILD_TYPE=RELEASE \
-D CMAKE_INSTALL_PREFIX=/usr/local \
-D OPENCV_EXTRA_MODULES_PATH=../../opencv_contrib-4.12.0/modules \
-D WITH_CUDA=ON \
-D WITH_CUDNN=ON \
-D WITH_TENSORRT=ON \
-D OPENCV_DNN_CUDA=ON \
-D CUDA_ARCH_BIN=8.7 \
-D WITH_TBB=ON \
-D WITH_V4L=ON \
-D WITH_GSTREAMER=ON \
-D WITH_OPENGL=ON \
-D BUILD_SHARED_LIBS=ON \
-D BUILD_opencv_python3=ON \
-D BUILD_TESTS=OFF \
-D BUILD_PERF_TESTS=OFF \
-D BUILD_EXAMPLES=OFF \
-D BUILD_DOCS=OFF \
-D BUILD_JAVA=OFF \
-D BUILD_opencv_java=OFF \
-D BUILD_opencv_js=OFF \
-D BUILD_opencv_apps=OFF \
..
make -j$(nproc)
sudo make install

整个过程分为两大步骤——编译和链接：
编译：把你写的 .cpp 源文件翻译成机器能看懂的二进制指令，生成 .o 目标文件。每个
.cpp 文件会各自编译成自己的 .o 文件，这些文件之间互不认识。
链接：把一堆 .o 目标文件和用到的库（比如 OpenCV）拼在一起，生成最终的可执行文
件。链接阶段会把你在 main.cpp 里调用的opencv库的函数（比如 cv::imread ） 跟
OpenCV 库里对应函数的真正地址对上号。
假设有一个最简单的 hello.cpp ：

#include <iostream>
int main() {
std::cout << "Hello, Robomaster!" << std::endl;
return 0;
}
逐步执行：

g++ hello.cpp -o hello
./hello

#输出hello robomaster!

但是如果项目内文件比较多了，举个例子：

my_project/
├── main.cpp
├── detector.cpp
├── tracker.cpp
├── serial.cpp
├── utils.cpp
├── detector.h
├── tracker.h
├── serial.h
└── utils.h
你需要手动敲：

g++ main.cpp detector.cpp tracker.cpp serial.cpp utils.cpp \
-I/usr/local/include/opencv4 \
-L/usr/local/lib \
-lopencv_core -lopencv_imgproc -lopencv_highgui -lopencv_video \
-I/usr/include/eigen3 \
-std=c++17 -O2 -g \
-o my_app
每次改一个文件就要把所有人全部重新编译一遍——慢且容易出错。
这就是CMake要解决的问题：自动管理编译规则、只重编译改过的文件、跨平台统一配置。
